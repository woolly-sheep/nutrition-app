# Nutrition App UI Design v0.5 追補 — 記録の管理（一覧・削除・過去日）

MVP実装完了後の実用ギャップを埋める追補。保存済みの記録を「見る・消す・
過去日に付ける」手段が v0.1 に未定義だったため、記録タブに追加する。
利用形態は本人のみ・ローカル継続（ユーザー決定 2026-07-17）のため
認証・マルチユーザー化は行わない。

## 1. 日付ナビゲーション（記録タブヘッダ）

- 日付ヘッダの左右に「← 前日」「翌日 →」（44px タップ・未来日へは進めない）
- 選択日は記録タブ全体（保存先・保存済み一覧・いつもの◯◯・候補）に適用
- 分析タブの「内訳になった食事を確認 →」「あとから追加 →」は
  該当日付き（`/meals?date=`）で遷移し、記録タブは該当日を開く

## 2. この日の記録（保存済み一覧）

「この食事に追加済み」（保存前・点線）の下に、**保存済み**セクションを置く。

- セクション見出し: 「この日の記録」
- 食事ごとに1カード: 食事区分ラベル + 品目（食品名+g）+ 合計推定kcal
- 確定値なので 5a の通常表示（実線・不透明）。点線プレビューと明確に区別
- 記録が無い日は「この日の記録はまだありません」を小さく表示

## 3. 記録の削除

- 各保存済みカードに「削除」（44px）
- タップで**その場で確認に変わる**（「削除する？ はい / いいえ」）。
  モーダルは使わない（1画面1つの問い・実装も薄く保つ）
- 削除は取り消し不可のため、確認なしの即時削除はしない
- 削除後は保存済み一覧・いつもの◯◯・候補を再取得（分析へ即反映）

## 4. safe wording 上の注意

- 削除・過去日記録はデータ管理機能であり、判定・助言の文言は追加しない
- 過去日の記録も「事実の記録」であり、日付以外の扱いは当日と同一
