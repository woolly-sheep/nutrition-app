# Nutrition App UI Design v0.4 追補 — 残り未決3件の解消

v0.1 §8 の未決のうち、残っていた3件（折りたたみ展開 UI・オンボーディング・
ユーザー名表示）を解消する追補。v0.1〜v0.3 の他の節はそのまま有効。

## 1. 不足折りたたみを開いた状態の UI

ホームの「他の不足 n件 ▾」を開いたときは、上位2件の RemainingCard
（主表現「あと◯◯」大）を繰り返さず、**コンパクト行**で表示する。

- 行の構成: 栄養素名 + 摂取/基準（補助）+ 「あと◯◯」（14px・強調なし）+
  細いバレットバー（高さ6px・5aと同じ塗り分け）
- 理由: v0.1 §2.4「数値は主表現1つ+補助1つまで」「1画面1つの問い」。
  展開してもホームの問いは「今日は何が足りないか」のまま、
  大きな数値の反復で密度を上げない
- 折りたたみサマリー行はタップ対象44px以上を維持（既存どおり）

## 2. オンボーディング（初回・注意事項と同意）

初回起動時の区分選択画面（ProfileSetup）を同意画面として本設計化する。
別画面は増やさない（初回の問いを1つに保つ）。

構成（上から順に）:

1. タイトル「はじめに」
2. このアプリができること 3点（6c と同文: 成分表で自動計算 /
   基準との比較 / 不足量の食品換算）
3. 注意事項: policy §5 の DISCLAIMER 全文（変更・省略禁止）
4. 基準値の区分選択（年齢帯 × 性別・既存）
5. CTA: 「**注意事項に同意して始める**」— 区分未選択時は無効

同意の扱い:

- MVP では**プロフィール保存の操作をもって同意とみなす**
  （CTA 文言で明示）。同意日時の永続化は行わない
- 認証・マルチユーザー導入時に consented_at の保存と再同意フローを
  設計する（POST /user-data:delete と同時期）

## 3. ユーザー名表示

**認証導入まで名前なし表現を正式決定とする。**
「はじめまして、佐藤さん」等の呼びかけは使わず、日付・事実のみで始める
（探索 6c の名前入り挨拶は認証設計確定後に再検討）。
