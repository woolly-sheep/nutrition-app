# Nutrition App UI Design v0.6 追補 — %E レンジバー（6b の脂質例を有効化）

decision-20260717-energy-ratio-dg により DG %E 判定が可能になったため、
v0.2 §3 で保留していた 6b の %E レンジバー表示を有効化する追補。

## 1. 分析タブ・DG超過セクションの %E 項目

たんぱく質・脂質・炭水化物が目標範囲（%E）を上回った場合:

- 補助数値: 「33 / 目標 20-30 %E」（摂取側は推定 %E・目標はseed表記のまま）
- 主表現: 「**目標範囲より +◯pt**」（単位は pt。g とは混同させない）
- **RangeBar**: 斜線帯=目標範囲（両端に墨色の境界線と % ラベル）・
  緑の内側バー=推定 %E・範囲上限を超えた分=墨色クロスハッチ（v0.2 §2 と同義）
- 食事別内訳は表示しない（食事ごとの %E は分母が異なり誤読を招くため）
- 文言・注記は既存の DG 超過と同一（SafeWordingService / DG_OVERAGE_NOTE）

## 2. 範囲を下回る場合・範囲内の場合

- 範囲内（within_goal）: ホームの「目標圏内 n項目」に含まれるのみ（新設表示なし）
- 下回る（below_goal）: MVP では専用表示を追加しない
  （不足の主導線は RDA/AI の 5b カード。過剰な問いを増やさない）

## 3. エネルギー摂取が算出できない日

%E は判定不能（unknown）のまま表示しない（従来どおり）。
