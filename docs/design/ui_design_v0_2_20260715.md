# Nutrition App UI Design v0.2 追補 — UL（耐容上限量）到達時の表示

v0.1 §8 未決事項「UL到達時の表示案」を解消する追補。v0.1 の他の節はそのまま有効。

## 1. 採用案: 7a 専用セクション分離型（ユーザー決定 2026-07-15）

分析タブ（日次）の**最上部**に、UL を上回った栄養素だけの独立セクションを置く。

- セクション見出し: 「耐容上限量(UL)に達した項目 n件」
- 該当 0 件の日はセクションごと非表示（常設しない）
- DG 超過セクション（v0.1 §4.3 の 6b）とは物理的に分離し、混在させない
  （6b 注記「UL が設定された栄養素が上限に達した場合のみ、別途明示します」への対応）

各項目の構成:

1. 栄養素名 + 補助数値「摂取 / UL 値」（例 `3,100 / UL 2,700 ug`）
2. しきい値バー: 摂取済み=緑塗り、**UL しきい値=墨色の縦線**、
   **UL を超えた分=墨色クロスハッチ**（下記 §2）
3. 文言: SafeWordingService `above_upper_limit` のラベル
   「耐容上限量(UL)を上回る推定値です」+ CONSULT_NOTE
   「参考情報です。必要に応じて専門家に相談してください。」（変更・追記禁止）
4. 食事別内訳（参考情報・任意）: 「内訳の上位: 昼食 ◯◯ug · 朝食 ◯◯ug」

## 2. カラー・セマンティクス追加（5a の拡張）

| 表現 | 意味 | 備考 |
|---|---|---|
| 墨色（text色 #202A2C）の縦線 | UL / DG のしきい値位置 | バー上のマーカー。データの塗りには使わない |
| 墨色クロスハッチ | UL / DG しきい値を超えた量 | **濃緑（=基準値100%超過）と意味を分離**。警告色（赤）は引き続き不使用。パターンなので色覚特性に依存しない |

- 濃緑: RDA/AI の基準値を超えた「摂った分」（良い/悪いの判断なし）
- 墨色クロスハッチ: UL/DG の上限・目標しきい値を超えた「事実の量」（判断なし）

## 3. 分析タブ（日次）のセクション順序（v0.1 §4.3 の確定）

```text
1. UL到達セクション（7a・0件なら非表示）
2. 基準値比較（RDA/AI バレットバー: 摂取=緑 / 残り=斜線 / 100%超過=濃緑）
3. 目標量(DG)を上回っているもの（6b・0件なら非表示）
   - 「目標 6.5g未満/日 → 8.2g（目標より +1.7g）」+ しきい値バー（§2）
   - 食事別内訳（参考情報）
   - 必須注記: DG_OVERAGE_NOTE（SafeWordingService 定義文を変更なしで使用）
4. 内訳になった食事を確認 →（記録タブへ）
5. 出典フッタ
```

- 脂質等の %E レンジバー（6b の脂質例）は MVP 対象外
  （DG %E レンジは判定対象外の既決定に従う。%E 判定導入時に追補する）
