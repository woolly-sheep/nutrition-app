# Nutrition App UI Design v0.7 追補 — 記録の編集（g修正・品目削除）

v0.5 の「この日の記録」カードに編集を追加する追補。
これまで修正は「削除→再登録」しかなかったギャップを埋める。

## 1. スコープ

- **できること**: 保存済み記録の各品目の g 修正・品目の削除
- **できないこと（意図的）**: 編集中の品目追加・食事区分/日付の変更。
  品目追加は通常の記録フロー（同じ日・同じ区分でもう1件保存）で足りる。
  区分・日付の付け替えは削除→再登録（頻度が低く、編集UIを複雑にしない）

## 2. UI（保存済みカードのインライン編集）

- カードの「削除」の隣に「編集」（44px）
- 編集モード: 品目ごとに `食品名 + [g入力] g + 取消`、下に
  「保存する / やめる」。モーダルは使わない（v0.5 §3 と同方針）
- 全品目を取り消すと保存不可（記録ごと消したい場合は削除を使う。
  ボタン無効化＋「品目が0件になる場合は記録の削除を使ってください」）
- 保存後は一覧・いつもの・候補を再取得（分析へ即反映）
- 編集は事実の修正でありプレビューではない → 点線表現は使わない

## 3. API

- `PUT /api/meals/{meal_id}`: body は `{ items: [{food_id, intake_g}] }`。
  検証は保存時と同一（既知food・0<g≤5000・1〜50品・値はエラーに含めない）。
  未知 id は 404。date / meal_type / meal_id / recorded_at は不変
