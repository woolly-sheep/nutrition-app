# decision-20260721-standard-weight-reference

## 決定

個数で数える食材（卵・食パン・バナナ等）を「個数」で記録できるようにするため、
**「標準重量リファレンス」を凍結 seed とは別のデータセットとして新設**する。

- 各食材に対し「1個(1枚/1本/1杯)あたりの標準重量(g)」を、出典付き・推定値として保持する
- ユーザーは食材ごとに自分の「1個あたりg」を設定でき（ユーザー所有データ）、
  設定値がある場合はそちらを優先する（B 案）
- 標準重量リファレンスは、ユーザー未設定時の**目安表示・初期値**としてのみ使う
- 記録・集計の実体は従来どおり **g**（個数 × 採用した1個あたりg → g で保存）。
  既存の API 契約・データモデルは変更しない

## これは公式 seed の改変ではない

絶対ルール「公式 MHLW/MEXT 値の手編集・再計算は禁止」は、**凍結 seed の成分値・
基準値・廃棄率そのもの**を対象とする。本件はそれらに一切触れない。

- 標準重量は成分表（八訂）には存在しない別種の情報（個体の物理重量）であり、
  成分値からの計算では導出できない
- したがって `seed/frozen/` は不変。標準重量は **別ファイルの参照データ**として持つ
- 既存の `seed/frozen/unit-conversion.json` は
  `review_status: use_explicit_grams_preferred` を明示しており、本決定はこの方針
  （信頼できる公的 g 値が無い食材はユーザー入力 g を優先）と整合する

## 出典ポリシー

標準重量は計算不能なため、必ず外部の実データを出典付きで登録する。

- **主：女子栄養大学出版部『調理のためのベーシックデータ』（最新版）** — 栄養士実務の
  事実上の標準。全表の転載はせず、必要な食材の値のみ出典明記で登録する（著作権配慮）
- **補：文部科学省 日本食品標準成分表(八訂) の廃棄率** — 「購入重量 → 可食部 g」換算の
  公式根拠として利用可（公式・Excel 公開）
- **暫定：公開されている自治体等の目安表**（値が揃うまでの暫定。正式データへ差し替え前提）
- 各エントリは `source`（出典）と `confidence`（信頼度）を必ず持つ

## データモデル（案）

凍結 seed とは分離した参照データ + ユーザー設定データの2層構成。

```text
standard-weight-reference（新設・参照データ / seed とは別ディレクトリ）
  food_id            凍結 seed の food_id を参照
  unit_label         "個" | "枚" | "本" | "杯" 等
  typical_weight_g   1単位あたりの標準重量（推定値）
  source             出典（例: 調理のためのベーシックデータ 第X版）
  confidence         high | medium | low

user portion（ユーザー所有データ / data/ 配下・seed 不変）
  food_id
  unit_label
  grams_per_unit     ユーザーが設定した1単位あたり g
```

採用優先順位: **ユーザー設定値 > 標準重量リファレンス > 未対応（g 入力のみ）**

## 文言（safe wording）

- 標準重量の表示は「一般的な目安: 1個 約◯g（推定・出典○○）」の形にし、
  **公式栄養値と誤認させない**
- ユーザー設定値は「あなたの設定値」と明示
- 断定・医療的示唆はしない（既存 SafeWordingService の方針に従う）

## 影響

- 記録画面: 対象食材で「個数 / g」の入力切替を追加。個数入力時は
  採用した1単位 g で換算して既存の記録フローに渡す
- 保存の実体は g のため、集計・分析・契約チェックへの影響は無い
- 標準重量リファレンス・ユーザー設定は段階導入可（未対応食材は現状の g 入力のまま）

## 未決事項

- 主要出典（『調理のためのベーシックデータ』最新版）の入手可否。未入手の場合は
  暫定ソースで主要食材（卵・食パン・バナナ・トマト等）から開始し、後で差し替える
- 対応食材の初期スコープ（どの食材まで標準重量を登録するか）
