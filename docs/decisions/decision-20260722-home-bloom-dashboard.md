# decision-20260722-home-bloom-dashboard

## 決定

ホーム（日次サマリー）を **栄養バランスの花（Bloom）** ダッシュボードに刷新する。
モチベーション設計を、safe wording と既存の配色ルールを守りつつ導入する。

## 構成

- **花（ヒーロー）**: 6つの「満たすほど咲く」栄養グループを花びらで表現。中心に当日の
  平均充足%。花は感情レイヤーで、正確な数値は下段（プログレッシブ・ディスクロージャー）
- **今週の庭**: 1日1輪の小さな花。記録した日は開花、空いた日は「明日のつぼみ」
  （減点でなく前向きな空状態）。習慣づけを罪悪感なく可視化
- **できていること**: 達成バッジ（ゴールド）を前面に（ピーク・エンド／ポジティブ）
- **あと少し**: 上位不足を細い棒＋「あと◯◯」。事実ベースの一手（食品候補）を併記
  （推奨ではない旨の notice を明示）

## 花びらに含める栄養素

「基準に到達＝良い（more is better）」の栄養素のみを花びらにする:
たんぱく質 / 食物繊維 / ビタミンA・D / ビタミンB・C(＋葉酸) / カルシウム・鉄 / カリウム・亜鉛。

**除外**: エネルギー・脂質・炭水化物（%Eバランスの範囲判定）、食塩（上限）。
これらは「多いほど良い」ではないため、花びらを伸ばす表現は誤解を招く。
→ 花びらの長さは常に「基準への近さ」を意味する（1色1意味の徹底）。

## ルール順守

- 進捗＝ティール、**達成＝ゴールドのみ**（花びらの到達／達成バッジ）。赤（警告色）不使用
- グループ平均が基準到達で花びらがゴールド（個別達成はバッジで表示）
- 断定・医療示唆なし。数値は推定・出典明記。候補は「推奨ではない」旨を明示

## データ

- 新APIなし。日次 `achieved[]/insufficient[]` の `percent_of_reference` を6グループに集約。
  週次 `nutrients[].daily` から各日の平均充足を算出。候補は既存 `/api/analysis/candidates`
- グループ集約・週の庭は純粋関数（`domain/analysis/nutrientGroups`・`weekGarden`）でテスト

## 影響

- ホームの見た目のみ刷新。分析タブ・記録・集計・API契約は不変
- %Eバランス（エネルギー・脂質等）はホームの花には出さない → 分析タブで従来どおり確認
