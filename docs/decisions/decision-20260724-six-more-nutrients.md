# decision-20260724: 栄養素を16→22種類に拡張する

- Status: accepted
- Date: 2026-07-24
- Context: 収録栄養素は16種類（うち判定対象12）。追加候補の選定基準は
  「公式基準値（MHLW DRI 2025）と成分値（MEXT 八訂 本表）が両方揃う」こと。
  ユーザー要望（美容・健康面での不足確認）を受けて再評価した。

## 決定

以下6種類を追加する。

| 栄養素 | code | 指標 | 摂取側の列 |
| --- | --- | --- | --- |
| マグネシウム | `magnesium_mg` | EAR / RDA | MG |
| 銅 | `copper_mg` | EAR / RDA / UL | CU |
| ビタミンE | `vitamin_e_mg` | AI / UL | TOCPHA（α-トコフェロール） |
| ナイアシン | `niacin_mgne` | EAR / RDA / UL | NE（ナイアシン当量） |
| ビタミンB6 | `vitamin_b6_mg` | EAR / RDA / UL | VITB6A |
| ビタミンB12 | `vitamin_b12_ug` | AI | VITB12 |

**効果（美容・健康への影響）を根拠にはしない。** アプリは事実提示のみを行う
（safe wording policy）ため、追加根拠は既存の選定基準に統一する。

### 単位の基準合わせ

摂取側と基準側で「同じものを数えている」ことを列レベルで担保した。

- ビタミンE は DRI が α-トコフェロールで算定されているため、`TOCPHA` を使う
  （`TOCPHB/G/D` は含めない）
- ナイアシンは DRI の単位が mgNE/日 のため、`NIA` ではなく `NE` を使う

### 比較しない上限値2件

公式値は転記するが、**摂取側と基準が異なるため判定しない**。

- **マグネシウム UL**: 表は通常の食品で「−」。350 mg/日 は通常の食品以外からの
  摂取に対する値（表脚注1）。`not_established` として保存し、脚注を
  `reviewer_note` に残す（鉄ULと同じ扱い）
- **ナイアシン UL**: 1セルにニコチンアミドとニコチン酸の2値が入り、どちらも
  mgNE ではない（表脚注3）。`"350 nicotinamide / 85 nicotinic_acid"` の条件付き
  文字列として保存し、判定層は `unknown(conditional_value)` を返す
  （鉄の月経条件分割と同じ扱い）。このため `officialValue` の条件付きパターンを
  月経限定から「数値＋条件 / 数値＋条件」の一般形に拡張した

ビタミンB6 UL（ピリドキシン相当量）と ビタミンE UL は数値のまま保存し、脚注を
`reviewer_note` に残す（ビタミンA UL がレチノール基準で数値保存されている先例に従う）。

## 出典と取得方法

- 成分値: 既存と同じ `seed/source/mext_ch2_honpyo.xlsx`（八訂 増補2023 第2章 本表）
- 基準値: 厚生労働省「日本人の食事摂取基準（2025年版）」策定検討会報告書 分割版PDF
  （`seed/source/mhlw_dri2025/`・**正誤表 令和7年3月25日 反映済みのHP掲載版**）
  - マグネシウム p.284 / 銅 p.347 / ビタミンE p.183 / ナイアシン p.236 /
    ビタミンB6 p.237 / ビタミンB12 p.238
  - 正誤表を確認し、上記6表の成人行に訂正がないことを検証済み

PDFは埋め込みフォントがCID符号化されており機械抽出でテキストが化けるため、
基準値150行は**表を目視で転記**した。転記結果は
`scripts/add_mhlw_references_2025.py` にリテラルとして保持し、代表値を
`tests/integration/added-nutrients.test.ts` で固定している。

## 影響

| 対象 | 変更 |
| --- | --- |
| `nutrient-amount.json` | 40,608 → **55,836行**（2,538食品 × 22）。2.3MB → **3.0MB** |
| `nutrient-reference.json` | 330 → **480行** |
| 既存データ | **1行も変更なし**。追記スクリプトは既存16種類をExcelから再抽出して一致検証してから追記する |
| `loadSeed` | NUTRIENT_META に6件追加（名称・単位） |
| `validateSeed` | 期待行数と NUTRIENTS_PER_FOOD 16→22 |
| 花（6グループ） | ビタA・D・E / ビタB・C / カル・鉄・Mg / カリ・亜鉛・銅 に再配分 |
| 逆引き検索 | 選択可能な栄養素に6件追加（12→18） |
| API契約 | **変更なし**（栄養素コードは値であって契約ではない） |

## 意図的に採らなかった選択肢

- **ヨウ素・セレン・クロム・モリブデン**: 日常の記録では動きが乏しく、ULが近い
  項目もあるため画面が「過剰」表示で埋まるリスクがある
- **ビオチン・パントテン酸**: AI のみで不足判定の意味が薄い
- **n-3系脂肪酸・飽和脂肪酸**: 本表ではなく脂肪酸成分表が必要で抽出コストが跳ね上がる
- **ナイアシン UL を数値で保存**: 摂取側 mgNE と基準が異なるため、比較すると
  事実と異なる「過剰」表示が出る
